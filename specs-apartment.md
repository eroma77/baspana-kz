# Спецификация полей для категории «Ищу квартиру»

### 1. География (Город и Район)

* **Город:** Выбор строго из существующей базы данных.
* **Район:** Динамическое поле. Выбор из списка районов данного города из БД.
> **Логика работы:** Если в выбранном городе в базе данных отсутствуют районы, поле «Район» автоматически становится **неактивным (disabled)** и **необязательным** для заполнения.

### 2. Возрастной диапазон

* **Диапазон:** от 16 до 50.
* **Логика выбора:** При клике на «От» выпадает список цифр от 16 до 50. При выборе значения (например, 18), поле «До» динамически перестраивается и показывает цифры **только от 18 до 50**. Валидация блокирует любые варианты, где «До» меньше «От».

### 3. Количественные параметры (Выпадающие списки)

* **Количество комнат:** Строгий выбор из списка: `1`, `2`, `3`, ..., `10+`.
* **Сколько человек нужно (для полного комплекта):** Выбор от `1` до `10+`.
* **Общее количество людей (которые будут жить):** Выбор от `1` до `10+`.

### 4. Ограничения по полу (Строгие литералы)

Разрешено использовать **только** три следующих варианта текста в интерфейсе и карточках:

* `Только парни`
* `Только девочки`
* `Не важно` (означает совместное проживание парней и девушек).

> **Запрещено:** подставлять слова «Все», «Любой» или дублировать знаки препинания.

### 5. Интеграция и Юридические параметры

* **Ссылка на 2GIS:** Текстовое поле ввода. Ожидает прямую валидную ссылку на адрес конкретного дома из приложения или веб-версии 2GIS.
* **Депозит:** Строгий выбор из двух вариантов: `Есть` или `Нет`.
* **Договор:** Строгий выбор из двух вариантов: `Есть` или `Нет`.

### 6. Ценовой диапазон (От / До) и Валидация

* Два поля для самостоятельного ручного ввода чисел (без готового выбора).
* **Нижний порог:** Минимальная цена — **30 000 тенге**. Ввод любого значения ниже (например, 29 999) блокируется, поле подсвечивается **красным цветом** и выводится сообщение об ошибке.
* **Верхний порог:** Максимальная цена — **900 000 тенге**. Любое число выше (например, 900 001) вызывает **красную подсветку** и ошибку валидации.

### 7. Медиа и Описание

* **Фотографии:** Добавление через кнопку «+». По клику открывается нативная галерея телефона.
* *Минимум:* **3 фотографии** (строгое условие публикации).
* *Максимум:* **5 фотографий** (мультивыбор в галерее блокируется при достижении 5 элементов).

* **Описание:** Текстовое поле. Полностью **необязательно** для заполнения.

### 8. Контакты и Публикация

* **Номер телефона:** Поле для ввода номера. Строго проверяется формат для связи через **WhatsApp**.
* **Кнопка отправки:** Содержит текст **«0 тенге»**. Означает бесплатную публикацию. При нажатии объявление мгновенно отправляется в базу данных и публикуется.

## Правила для отображения в компонентах (`ListingCard` и др.)

При рендере карточки для категории «Ищу квартиру» тексты параметров должны в точности соответствовать выбранным полям из спецификации выше. Никаких генераций текста от себя — только чистые данные по макету. Все правила сохранены для дальнейшей работы.
