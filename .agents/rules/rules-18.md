# 📜 MASTER PROJECT RULES & SPECIFICATION (ROOMMATE APP KAZAKHSTAN)
**Стек:** Next.js (App Router), React, Tailwind CSS, TypeScript, Zustand, Supabase.
**Главная цель:** Создание стабильного, "твердого" веб-приложения со строгим следованием дизайну Figma, исключающим любые галлюцинации ИИ.

---

## ГРУППА 1.0: ТЕХНИЧЕСКИЙ СТРОГИЙ СТЕК И АРХИТЕКТУРА
1.1. Маршрутизация: Использовать исключительно Next.js App Router. Структура папок должна отражать разделы нижнего меню.
1.2. Server/Client компоненты: По умолчанию все файлы — Server Components. Директива `'use client'` применяется точечно только для интерактивных форм, кнопок и свитчей тем.
1.3. Логика базы данных и авторизации: 
    - В качестве бэкенда и БД используется строго **Supabase**.
    - Авторизация реализована через **нативный Google OAuth (Supabase Auth)**. 
    - При клике на "Войти через Google" приложение обязано вызывать официальное окно выбора аккаунтов Google (`supabase.auth.signInWithOAuth({ provider: 'google' })`). Никаких кастомных кодов подтверждения или паролей. Сессия обрабатывается автоматически.
1.4. Управление стейтом: Глобальное состояние темы, сессии пользователя и Избранного (Корзины) управляется через легковесный **Zustand** с middleware `persist` для сохранения избранного в `localStorage`.
1.5. Строгий TypeScript: Полный запрет на использование типа `any` или аннотаций `// @ts-ignore`. Каждая сущность (Объявление, Пользователь, Тариф) обязана иметь жесткий `interface` или `type`.

## ГРУППА 2.0: ГЛОБАЛЬНАЯ ЛОГИКА, АВТОРЕЗАЦИОННЫЙ ЩИТ (AUTH GUARD) И НАВИГАЦИЯ
2.1. Переключатель режимов (Top Mode Toggle):
    - Черная капсула сверху с кнопками `ищу квартиру` и `ищу соседа`. Активное состояние — белая скругленная пилюля с жирным черным текстом. Переключение полностью меняет контекст выдачи объявлений и формы добавления/редактирования.
2.2. Нижнее меню (Bottom Navigation Bar): Содержит 5 фиксированных вкладок (Лента, Избранное, Добавить (+), Просмотрено, Профиль). Активное состояние вкладки выделяется синим залитым контуром, остальные — серые/белые.
2.3. **Жесткий Авторизационный Щит (Auth Wall):**
    - Если пользователь **НЕ авторизован** (в стейте Supabase отсутствует активная сессия), при попытке совершить любое из следующих действий:
        1. Нажатие на кнопку "Ватцап" (на любой карточке или странице).
        2. Нажатие на иконку "Сердце" (Добавить в Избранное/Корзину).
        3. Нажатие на центральную кнопку добавления объявления `+`.
        4. Нажатие на кнопку "Оплатить" или "Редактировать".
    - Система обязана заблокировать действие, прервать выполнение и автоматически перенаправить пользователя на вкладку **Профиль ("мой кабинет")**, требуя пройти авторизацию через Google. Действие не будет выполнено, пока сессия не станет активной.

## ГРУППА 3.0: УМНАЯ ВАЛИДАЦИЯ И ЭРГОНОМИКА ФОРМ (UX UI RULES)
3.1. **Абсолютный запрет на ручной ввод в селектах:** Все поля, требующие выбора параметров (Город, Район, Пол, Комнатность, Срок, Депозит, Договор), должны быть реализованы исключительно как кастомные выпадающие списки (Dropdowns). Любая возможность ручного ввода букв с клавиатуры в этих полях должна быть полностью заблокирована (`readOnly` или кастомный UI без элемента input).
3.2. **Умная валидация ошибок (Красная подсветка):**
    - Если пользователь пытается нажать кнопку "Отправить" / "Готово", не заполнив обязательные поля или заполнив их неверно, эти поля обязаны мгновенно получить **красную рамку/границу (border-red-500)**.
    - Как только пользователь начинает исправлять ошибку (вводить корректные данные или выбирать пункт из дропдауна), красная подсветка с этого конкретного поля обязана **автоматически исчезнуть** в реальном времени, возвращая дефолтный серый/белый контур.
3.3. **Автоматический разделитель тысяч (Smart Currency Input):**
    - В полях ввода бюджета/депозита ("От" и "До") при вводе чисел ИИ обязан на лету форматировать отображение текста, добавляя пробел в качестве разделителя тысяч. 
    - Пример: пользователь вводит `10000` -> инпут динамически отображает `10 000`. Пользователь вводит `150000` -> инпут отображает `150 000`. При этом в стейт и бэкенд данные должны уходить в виде чистого числа (`number`), без пробелов.
3.4. **Жесткие ограничения типов ввода:**
    - Текстовые поля (например, Описание): Блокировать ввод подозрительных системных спецсимволов.
    - Числовые поля ("От", "До"): Строго `input type="number"`, блокировать ввод символов `e`, `+`, `-`, точек и запятых. Разрешены только целые положительные числа (`>= 0`).

## ГРУППА 4.0: ЖЕСТКАЯ СТАБИЛИЗАЦИЯ И ТЕМЫ (ANTI-SHAKE & THEMES)
4.1. Запрет случайного зума: ИИ обязан жестко зафиксировать мета-тег, отключающий pinch-to-zoom и double-tap zoom в мобильных браузерах:
    `<meta name="viewport" content="width=device-width, initial-scale=1.0, maximum-scale=1.0, user-scalable=no">`
4.2. Политика отсутствия шатания (Zero-Wiggle): Полный запрет горизонтального скролла на уровне CSS: `html, body { overflow-x: hidden; }`. Все компоненты должны стоять монолитно, размеры отступов и кнопок зафиксированы, верстка не имеет права сдвигаться ни на пиксель при кликах.
4.3. Кадрирование изображений: Все аватарки и фото жилья обязаны иметь свойство `object-fit: cover; object-position: center;`. Искажение пропорций лиц или интерьеров запрещено.
4.4. Логика Темной Темы: Реализация через класс `dark` на теге `html`. Каждому элементу прописывать парные стили (например, `bg-white dark:bg-zinc-900 border-gray-200 dark:border-zinc-800`).

5.1. Режим «Ищу квартиру»: Карточка содержит фото жилья, плашку "В ТОПЕ" (если премиум), цену, дату публикации, двухколоночную сетку из 8 тегов с иконками, синюю кнопку «Ватцап» (55% ширины, генерирует wa.me ссылку) и серую кнопку «2 гис» (40% ширины, открывает карту).

## ГРУППА 6.0: СТРУКТУРА СТРАНИЦ ДЕТАЛИЗАЦИИ И МОДАЛОК
6.1. Детализация Квартиры: Закрепленные кнопки «Ватцап» и «2 гис» делят экран 50/50 сверху под ценой. Слайдер вмещает строго от 3 до 5 фото. Количество индикаторных точек внизу фото строго равно количеству загруженных изображений.
6.2. Детализация Соседа: Сверху горизонтальный блок с квадратным селфи автора, ценой, датой и сердцем. Кнопка «Ватцап» идет на 100% ширины перед описанием. **Кнопка «2 гис» отсутствует.** Лимит фото в слайдере — **максимум 3 фото**, минимального порога нет.
6.3. Модалки фото: Полноэкранный оверлей с размытием заднего плана, нативными кнопками `‹` и `♥` поверх фото, сохраняющий лимиты пагинации (5 точек для квартир, 3 точки для соседей).

## ГРУППА 8.0: ЛИЧНЫЙ КАБИНЕТ, КОРЗИНА И ТАРИФЫ ПРОДВИЖЕНИЯ
8.1. Авторизованный профиль: Синяя кнопка входа исчезает. Появляется ярко-розовая кнопка `Выйти` (`#ff3b57`). Появляется разделительный баннер `Мои объявления` со счетчиком количества. На карточках объявлений вместо «Ватцап» рендерится вертикальный стек кнопок: `Редактировать` (синяя), `Рекламировать` (синяя), `Удалить` (розовая в Light, темно-серая в Dark).
8.2. Экран продвижения (Тарифы):
    - Кнопка назад `‹` и текст `рекламировать` в черной капсуле. Справа только свитч темы.
    - Стек тарифов:
        1. `3 дня в ТОПЕ` (В Light — белый фон с синим контуром; в Dark — полностью синий фон. Бейдж `190 ₸`, старая цена `390 ₸` зачеркнута красным).
        2. `7 дней в ТОПЕ` (В Light — белый фон с серым контуром; в Dark — полностью синий фон. Бейдж `490 ₸`, старая цена `690 ₸` зачеркнута красным).
        3. `30 дней в ТОПЕ` (В Light — фиолетовый фон и контур, бейдж `590 ₸`; в Dark — полностью фиолетовый фон, бейдж прозрачный светлый `590 ₸`. Старая цена `790 ₸` зачеркнута красным).
    - Нижняя кнопка `Оплатить`: Малинового цвета на 100% ширины с круглой иконкой внутри.
8.3. Корзина (Избранное): Шапка `корзина` в черной капсуле. Кнопка сердца на карточках становится **полностью залитой ярко-красным цветом**. Повторный клик удаляет карточку из экрана.
